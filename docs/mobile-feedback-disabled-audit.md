# Mobile Disabled Feedback Audit

Last updated: 2026-06-10

This log tracks disabled-control feedback and mobile tooltip behavior. The goal is to make unavailable controls explain their reason on hover, focus, and touch, and to keep feedback components usable on mobile.

## Audit Rules

- Disabled controls must expose a reason with GunjoUI `Tooltip` behavior.
- Native disabled elements must be wrapped in an enabled trigger when the tooltip needs to attach to the disabled target.
- Icon-only, ambiguous, or stateful feedback controls need accessible labels and touch-accessible tooltip feedback.
- Docs examples must keep preview and code aligned when a disabled reason is shown.
- Primitive components may expose disabled styling only; the consuming composition or docs example is responsible for the user-facing reason.
