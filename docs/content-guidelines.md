# GunjoUI Content Guidelines

Last updated: 2026-05-23

Tracking issues:

- #234 `docs: define GunjoUI content guidelines and message catalog`
- #235 `メッセージ一覧を実運用レベルまで拡充する`
- #236 `既存コンポーネント文言を文言ガイド基準で棚卸しする`
- #237 `文言ガイドを docs テンプレートの確認項目に組み込む`
- #239 `docs: 画像の代替テキストルールを文言ガイドへ組み込む`

Public docs page: `/docs/guidelines/writing`

GunjoUI の UI コピー、説明文、ボタンラベル、ツールチップ、エラー、空状態、トーストなどの文言を全体で揃えるためのガイドラインです。

GunjoUI は「日本語でも崩れない UI」を重要な品質として扱うため、文言は翻訳の後工程ではなく、コンポーネント設計・ドキュメント設計・パターン設計と同じレイヤーで確認します。

## 目的

- GunjoUI の docs / patterns / examples で使う UI コピーの判断基準を揃える。
- 日本語と英語で意味、粒度、トーンがずれないようにする。
- ボタン、ツールチップ、無効化理由、エラー、空状態などを毎回その場で考えないようにする。
- コンポーネント詳細ページのレビュー時に、見た目だけでなく文言品質も確認できるようにする。

## 対象範囲

対象:

- コンポーネントの表示文言
- ドキュメントページの説明文
- パターンページ内の UI コピー
- ボタン、リンク、タブ、メニュー、トースト、ツールチップ、空状態、確認ダイアログ
- 無効化理由、バリデーション、エラー、成功、読み込み中のメッセージ

対象外:

- 各プロダクト固有のマーケティングコピー
- 実運用サービスの法務文言
- 業界特化パターンの専門文言
- ユーザー企業ごとのブランドボイス

ただし、対象外の文言でも GunjoUI のサンプルとして掲載する場合は、このガイドラインの最低限の読みやすさとレイアウト耐性を満たします。

## 基本トーン

GunjoUI の文言は、短く、具体的で、操作につながることを優先します。

- 明確: 何が起こるのか、何をすればよいのかがわかる。
- 穏当: ユーザーを責めない。
- 簡潔: ボタンやラベルは短く、説明は必要な時だけ足す。
- 一貫: 同じ操作には同じ言葉を使う。
- 実用: 見た目のための文言ではなく、判断や操作を助ける文言にする。

避ける表現:

- `エラーが発生しました` だけで終わる。
- `実行`, `処理`, `適用` など、対象が曖昧な動詞を単独で使う。
- 日本語ページで説明文だけ英語になる。
- `FormMessage`, `destructive`, `variant` などの実装語を、ユーザー向け説明にそのまま混ぜる。
- ボタンに長い説明を入れる。

## 日本語コピーのルール

### ラベル

- 名詞または短い名詞句にする。
- 画面上で繰り返すラベルは、同じ言葉に揃える。
- 技術用語を使う場合は、必要なら補助テキストで説明する。

例:

| 用途 | 推奨 | 避ける |
| --- | --- | --- |
| ファイル名 | `ファイル名` | `タイトル` と混在 |
| 状態 | `状態` | `ステータス` と混在 |
| 件数 | `表示件数` | `Rows per page` |

### ボタン

- 操作結果がわかる動詞にする。
- 破壊的操作は対象を含める。
- アイコンだけのボタンは、必ずツールチップと `aria-label` を揃える。

例:

| 用途 | 日本語 | 英語 |
| --- | --- | --- |
| 保存 | `保存` | `Save` |
| キャンセル | `キャンセル` | `Cancel` |
| 削除 | `削除` | `Delete` |
| 完全削除 | `完全に削除` | `Delete permanently` |
| 共有 | `共有` | `Share` |
| コピー | `コピー` | `Copy` |
| コードコピー | `コードをコピー` | `Copy code` |
| AI用仕様コピー | `AI用仕様をコピー` | `Copy spec for AI` |

### 説明文

- 1文で足りるなら1文にする。
- 実装語ではなく、ユーザーが判断できる言葉にする。
- 先に目的、必要なら次に制約を書く。

例:

- 推奨: `保存すると、変更内容が一覧に反映されます。`
- 避ける: `保存後にテーブルへ反映されます。`

### ツールチップ

- アイコンだけの操作、状態が変わる操作、破壊的操作、無効化された操作には必ず付ける。
- 操作できる場合は「何が起こるか」を書く。
- 無効化されている場合は「なぜできないか」と「どうすればできるか」を書く。

例:

| 状態 | 日本語 | 英語 |
| --- | --- | --- |
| 開く | `カレンダーを開く` | `Open calendar` |
| 閉じる | `カレンダーを閉じる` | `Close calendar` |
| 選択前 | `選択します` | `Select` |
| 選択済み | `選択を外します` | `Deselect` |
| 無効化 | `権限がないため変更できません。管理者に依頼してください。` | `You do not have permission to change this. Ask an admin.` |

## メッセージパターン

### 空状態

空状態は「何がないか」と「次に何をすればよいか」を伝えます。

| 要素 | 日本語 | 英語 |
| --- | --- | --- |
| タイトル | `まだ{対象}がありません` | `No {object} yet` |
| 説明 | `{操作}すると、ここに表示されます。` | `{Action} to show it here.` |
| 操作 | `{対象}を追加` | `Add {object}` |

例:

- `まだファイルがありません`
- `アップロードすると、ここに表示されます。`
- `ファイルをアップロード`

### 成功

成功メッセージは短く、完了した事実を伝えます。

| 用途 | 日本語 | 英語 |
| --- | --- | --- |
| 保存 | `保存しました` | `Saved` |
| コピー | `コピーしました` | `Copied` |
| 削除 | `{対象}を削除しました` | `{Object} deleted` |
| 更新 | `{対象}を更新しました` | `{Object} updated` |

### エラー

エラーは「何ができなかったか」と「次にできること」を分けて書きます。

| 要素 | 日本語 | 英語 |
| --- | --- | --- |
| タイトル | `{操作}できませんでした` | `Could not {action}` |
| 説明 | `時間をおいてもう一度お試しください。` | `Try again in a moment.` |
| 入力エラー | `{項目名}の形式を確認してください。` | `Check the {field} format.` |

避ける表現:

- `失敗しました` だけで終わる。
- 原因が不明なのに原因を断定する。
- ユーザーの入力ミスを責める。

### 警告

警告は、操作を止めるほどではないが注意が必要な時に使います。

| 用途 | 日本語 | 英語 |
| --- | --- | --- |
| 未保存 | `未保存の変更があります` | `You have unsaved changes` |
| 制限 | `一部の項目は変更できません` | `Some fields cannot be changed` |
| 容量 | `容量が上限に近づいています` | `Storage is almost full` |

### 破壊的操作の確認

削除、完全削除、取り消せない操作は、実行前に確認します。

| 要素 | 日本語 | 英語 |
| --- | --- | --- |
| タイトル | `{対象}を削除しますか？` | `Delete {object}?` |
| 説明 | `この操作は取り消せません。` | `This action cannot be undone.` |
| キャンセル | `キャンセル` | `Cancel` |
| 実行 | `削除` | `Delete` |

完全削除の場合は、ボタンも `完全に削除` にします。

### 無効化

無効化された UI は、必ず理由を説明します。

| 状態 | 日本語 | 英語 |
| --- | --- | --- |
| 権限なし | `権限がないため変更できません。` | `You do not have permission to change this.` |
| 条件不足 | `{条件}を設定すると操作できます。` | `Set {condition} to enable this.` |
| 処理中 | `処理中のため操作できません。` | `This is unavailable while processing.` |
| 上限到達 | `上限に達したため追加できません。` | `You have reached the limit.` |

### 読み込み中

読み込み中は、対象がわかるようにします。

| 用途 | 日本語 | 英語 |
| --- | --- | --- |
| 一般 | `読み込み中` | `Loading` |
| 対象あり | `{対象}を読み込んでいます` | `Loading {object}` |
| 保存中 | `保存中` | `Saving` |
| 送信中 | `送信中` | `Submitting` |

## メッセージ一覧の拡張方針

公開ページ `/docs/guidelines/writing` のメッセージ一覧は、単なるラベル集ではなく「ラベル」「表示メッセージ」「確認ポイント」を分けて管理します。

優先的に増やすカテゴリ:

- 入力と検証: 必須、形式、文字数、範囲、重複、一致しない。
- ファイル操作: アップロード、ダウンロード、容量超過、形式未対応、共有リンク。
- 一覧とテーブル: フィルター、検索結果なし、並び替え、ページ移動、一括選択。
- 認証と権限: サインイン失敗、セッション切れ、権限申請、招待期限切れ。
- 状態通知: 保存、送信、同期、オフライン、処理中、完了、失敗。
- 画像と代替テキスト: 装飾画像、内容を伝える画像、商品・素材画像、画像ボタン、グラフ、地図。

追加ルール:

- 新しい文言を docs / patterns に入れる時は、既存カテゴリに同じ意図の文言がないか確認する。
- 既存カテゴリにない場合は、実装 PR 内で一覧へ追加するか、#235 の追加候補として残す。
- ラベルだけで意味が足りない場合は、本文メッセージに対象、理由、次の行動を入れる。
- 英語は日本語の直訳ではなく、同じ操作意図と同じ情報量になるように書く。

## 画像と代替テキスト

画像の `alt` は、画像が UI の中で何を担っているかを説明する文言です。Img、ImagePreview、AssetCard、Avatar、グラフ、地図など、画像を扱うコンポーネントは文言レビューの対象にします。

| 役割 | ルール | 例 |
| --- | --- | --- |
| 装飾画像 | 見た目の装飾だけで、内容や操作を伝えない画像は `alt=""` にする | 背景装飾、区切り用の画像 |
| 内容を伝える画像 | 見た目ではなく、ユーザーに伝える内容を書く | `オフィスデスクの写真` |
| 商品・素材画像 | ファイル名だけにせず、対象名や内容がわかる言葉にする | `ノートパソコンの商品写真` |
| 画像ボタン | 操作は画像の `alt` ではなく、ボタンの `aria-label` とツールチップで説明する | `拡大表示する` / `画像を閉じる` |
| グラフ・地図 | `alt` だけで完結させず、近くに要約や表を置く | `東京都インシデントは新宿区が最多です。` |

避ける表現:

- `画像`
- `写真`
- `イメージ`
- `サンプル画像`
- ファイル名だけの `alt`

ただし、画像自体がユーザーに何も伝えない装飾であれば、無理に説明を書かず `alt=""` にします。画像の操作性は `alt` ではなく、操作するボタン、リンク、ツールチップ、補助テキストで説明します。

参考文献:

- [W3C WAI: Decorative Images](https://www.w3.org/WAI/tutorials/images/decorative/)
- [W3C WCAG Technique H67](https://www.w3.org/WAI/WCAG21/Techniques/html/H67.html)
- [MDN: HTMLImageElement alt](https://developer.mozilla.org/docs/Web/API/HTMLImageElement/alt)

## 長さとレイアウト耐性

日本語は英語より短く見える場合もありますが、UI 上では折り返しや禁則の影響で崩れやすい箇所があります。

| 場所 | 推奨長 | ルール |
| --- | --- | --- |
| ボタン | 2-8文字程度 | 12文字を超える場合は説明を外に出す |
| アイコンツールチップ | 8-24文字程度 | 1行で読める長さを優先 |
| 空状態タイトル | 10-20文字程度 | 対象を具体的にする |
| 空状態説明 | 30-60文字程度 | 1-2文まで |
| トーストタイトル | 6-18文字程度 | 完了・失敗を短く伝える |
| エラー説明 | 30-80文字程度 | 回復方法を含める |
| 表のセル | 折り返し可 | 型や長い値は折り返しまたは横スクロール |

レビュー時は、デスクトップ、タブレット、スマホ幅で以下を確認します。

- ボタンラベルが段落ちしていない。
- 説明文が操作ボタンを押し出していない。
- ツールチップやポップオーバーが見切れていない。
- 日本語と英語で意味が変わっていない。
- `default`, `variant`, `FormMessage` などの実装語がユーザー向け文に混ざっていない。

## コンポーネントドキュメントへの適用

コンポーネント詳細ページでは、次の項目を文言レビュー対象にします。

- タイトルと英語名
- 説明文
- Preview 内のラベル、ヘルプ、エラー、トースト
- 状態とバリエーションのタイトルと説明
- Props の説明
- Usage の説明
- 使用コンポーネント / 関連コンポーネント
- コード例内のサンプル文言
- 画像の `alt`、画像ボタンの `aria-label`、画像周辺の要約テキスト
- Copy spec for AI とコードコピーのフィードバック

チェックリスト:

- [ ] 日本語ページで英語だけの説明が残っていない。
- [ ] 英語ページで日本語だけの説明が残っていない。
- [ ] ボタンやツールチップの文言が操作結果を示している。
- [ ] 無効化 UI に理由がある。
- [ ] エラーに回復方法がある。
- [ ] 破壊的操作に確認がある。
- [ ] 画像の代替テキストが、装飾、内容、操作、データ可視化の役割に合っている。
- [ ] 日本語の長さでレイアウトが崩れていない。
- [ ] 英語の長さでレイアウトが崩れていない。

## 実装方針

短期:

- この文書をコンポーネント docs 横展開の確認基準に追加する。
- 新規・修正コンポーネントのレビュー時に、文言レビューを必須にする。
- docs / patterns の目立つ英語混在を修正する。

中期:

- よく使う文言を `app/lib/translations.ts` などの共通辞書に集約する。
- コンポーネントごとのラベル props を整理し、デフォルト文言を日本語と英語で揃える。
- 空状態、トースト、確認ダイアログ、無効化理由のサンプルを増やす。
- 既存コンポーネント docs を文言ガイド基準で棚卸しし、英語混在、片言表現、理由のない無効化、回復方法のないエラーを修正する。
- docs テンプレートの確認項目に文言ガイド準拠を組み込み、カテゴリ sweep ごとの完了条件にする。

長期:

- 公開ドキュメントとして `/docs/guidelines/writing` のようなページに展開する。
- component docs audit に文言チェックを組み込む。
- UI コピーの lint または検出スクリプトを検討する。
